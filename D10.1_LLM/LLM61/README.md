# Task 10.1D — Further Improving LLM-Enhanced Learning Assistant App

A Jetpack Compose Android app that extends the Task 6.1D Learning Assistant with three new screens — **Profile**, **History**, and **Upgrade** — and adds persistent quiz history, subscription-based tier purchasing (via Stripe), profile sharing, and a third Gemini-powered AI feature (weak-area analysis). Built in Kotlin with Material 3.

## Features
- Eight-screen flow: Login → Interests → Home → Quiz → Results → Profile → History → Upgrade
- Auth0 OAuth handles authentication (replaces local signup)
- Interest selection: pick up to 10 of 15 topics; edit anytime from Home
- Personalised Home screen: task count scales with subscription tier
- **AI Hint** on every quiz question — powered by Google Gemini
- **AI Explanation** on every result card, tailored to correct vs. incorrect answers
- **AI Weak-Area Summary** on Profile — Gemini analyses your recent incorrect answers
- Prompt sent to the model is shown (collapsible) alongside the response
- Quiz History stored in Room database with expandable per-question detail
- Three paid subscription tiers via Stripe; tier affects question count and AI extras
- Material 3 theme with `animateContentSize` cards and 400 ms slide + fade route transitions

## LLM Features

Three Gemini-powered utilities satisfy the task's "further improving with LLMs" requirement.

### 1) AI Hint — Quiz Screen
- **Files:** [QuizScreen.kt](app/src/main/java/com/example/llm61/screens/QuizScreen.kt) + [QuizViewModel.kt](app/src/main/java/com/example/llm61/viewmodel/QuizViewModel.kt)
- Prompt instructs Gemini **not to reveal the answer** — gives a 1–2 sentence encouraging hint
- States: Idle → Loading (`Generating hint…`) → Success → Error (`Try Again`)
- Success shows collapsible `📨 Prompt sent to AI` card + `💡 AI Hint` response card

### 2) Explain with AI — Results Screen
- **Files:** [ResultsScreen.kt](app/src/main/java/com/example/llm61/screens/ResultsScreen.kt) + [ResultsViewModel.kt](app/src/main/java/com/example/llm61/viewmodel/ResultsViewModel.kt)
- Prompt adapts by correctness: reinforcement (2–3 sentences) when right; correction + encouragement when wrong
- Friendly error messages for timeouts, HTTP 429 rate-limits, and offline (`Unable to resolve host`)
- Success shows `🤖 AI Explanation` card

### 3) Summarize Weak Areas — Profile Screen ✨ NEW
- **Files:** [ProfileScreen.kt](app/src/main/java/com/example/llm61/screens/ProfileScreen.kt) + [ProfileViewModel.kt](app/src/main/java/com/example/llm61/viewmodel/ProfileViewModel.kt)
- Builds a learning-coach prompt from the 15 most-recent incorrect questions in Room
- Identifies 2–3 topic areas with study recommendations
- When no incorrect answers yet: returns an encouraging message instead
- Shows `🧠 AI Summary` card with the same collapsible prompt display

### Shared Infrastructure
- Sealed state [LlmUiState.kt](app/src/main/java/com/example/llm61/network/LlmUiState.kt): `Idle`, `Loading`, `Success(prompt, response)`, `Error(prompt, message)`
- Reusable `PromptAndResponseDisplay` composable (defined in `QuizScreen.kt`) renders the collapsible prompt + response for all three utilities
- Network: [GeminiService.kt](app/src/main/java/com/example/llm61/network/GeminiService.kt) + [GeminiRepository.kt](app/src/main/java/com/example/llm61/network/GeminiRepository.kt) + [GeminiModels.kt](app/src/main/java/com/example/llm61/network/GeminiModels.kt)
- Model: `gemini-flash-latest` via `v1beta/models/gemini-flash-latest:generateContent`

## Quiz History

All quiz attempts are persisted in a Room database so they survive app restarts.

- **QuizAttemptEntity** — topic, score counts, timestamp, linked to the logged-in user
- **QuestionAttemptEntity** — per-question text, all options (JSON), user's selection, correct index, pass/fail flag
- **QuizHistoryRepository** — exposes `Flow<List<QuizAttemptEntity>>` for the History screen; saves attempts from `ResultsViewModel`
- Attempts are saved exactly once per Results visit (guarded by `hasSaved` flag in `ResultsViewModel`)

---

## Subscription Tiers & Payments

Payments are processed via **Stripe PaymentSheet** backed by a Vercel serverless function.

- `PaymentRepository.createPaymentIntent()` — calculates proration from remaining billing days; returns a Stripe `clientSecret`
- `PaymentRepository.restoreSubscription()` — queries Stripe charge history; returns the highest tier ever paid and timestamps
- `UserEntity.tier` / `cancelledAt` / `scheduledTier` — persisted locally in Room; updated after successful payment or cancellation
- **Cancellation** sets `cancelledAt`; access continues until the billing period ends, then tier reverts to Free
- **Scheduled downgrade** sets `scheduledTier`; transition happens at period end
- **Resume** clears both fields immediately

## How to Run

1. Clone the repo and open the **LLM61** folder as an Android Studio project (JDK 17, Android SDK 36)
2. Obtain a **Gemini API key** from Google AI Studio
3. Create / open `local.properties` and add:
   ```
   GEMINI_API_KEY=your_gemini_key_here
   ```
4. Configure **Auth0**: create a Native application in the Auth0 dashboard and update `auth0Domain` / `auth0ClientId` in `strings.xml` (or `local.properties` if wired via BuildConfig)
5. Configure **Stripe**: set your publishable key and point `PaymentApi` at your Vercel backend URL
6. Sync Gradle, then run the `app` module on an emulator or physical device (Android 7.0+ / API 24+)
7. The app requires the `INTERNET` permission (declared in `AndroidManifest.xml`)
