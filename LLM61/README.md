# Task 6.1D — LLM-Enhanced Learning Assistant App

A Jetpack Compose Android app that gives students topic-based quizzes with **Gemini-powered AI hints** while attempting questions and **AI explanations** on the results screen. Built in Kotlin with Material 3.

---

## Features
- Six-screen flow: Login → Signup → Interests → Home → Quiz → Results
- Interest selection: pick up to 10 of 15 topics; edit anytime from Home
- Personalised Home screen: 3 task cards drawn from the user's interests (or random topics if none chosen)
- **AI Hint** button on every quiz question — powered by Google Gemini
- **AI Explanation** button on every result card, tailored to correct vs. incorrect answers
- Prompt sent to the model is shown (collapsible) alongside the response — satisfies the task brief's "show prompt + response" requirement
- Full loading / error handling with a `Try Again` button on failures
- Material 3 theme with `animateContentSize` cards and 400 ms slide + fade route transitions
- Consistent Back navigation — system Back works on every screen; no dead ends

## Screens

### 1) Login — [LoginScreen.kt](app/src/main/java/com/example/llm61/screens/LoginScreen.kt)
- Username + password fields with inline validation (blank fields show an error message)
- `Login` button + `Need an Account?` text link to Signup

### 2) Signup — [SignupScreen.kt](app/src/main/java/com/example/llm61/screens/SignupScreen.kt)
- Username, email, confirm email, password, confirm password, phone
- Error messages surface inline (`Emails don't match`, `Passwords don't match`, etc.)

### 3) Interests — [InterestsScreen.kt](app/src/main/java/com/example/llm61/screens/InterestsScreen.kt)
- 15 topics in a 2-column grid of `FilterChip`s
- Bottom bar shows `"{n} of 10 selected"` with a `Next` button (enabled only when at least 1 topic is picked)
- 10-topic cap enforced in `UserViewModel.toggleInterest`
- Back arrow when entered from Home; `Skip` text button when entered from Signup

### 4) Home — [HomeScreen.kt](app/src/main/java/com/example/llm61/screens/HomeScreen.kt)
- Top bar: `Hello, {username}`
- Actions: pencil icon to edit interests, `Logout` button
- Body: "You have N task(s) due" + 3 generated `TaskCard`s — each tagged `✨ Generated Task` and opens a Quiz for that topic

### 5) Quiz — [QuizScreen.kt](app/src/main/java/com/example/llm61/screens/QuizScreen.kt)
- Top bar: `✨ Generated by AI` / `Quiz: {topic}`
- 3 question cards, each with 4 radio-button options
- `Get AI Hint` button on every card (see **LLM Features** below)
- Bottom bar: `"{n} of 3 answered"` + `Submit` (enabled only when all 3 are answered)

### 6) Results — [ResultsScreen.kt](app/src/main/java/com/example/llm61/screens/ResultsScreen.kt)
- Top bar: `✨ Answered by AI` / `Your Results`
- Score card: `Topic: {topic}` + large `{score} / {total}` + `Correct Answers` label
- One result card per question with a check-circle or cancel icon, the question text, `Your answer: …`, and (when wrong) `Correct answer: …`
- `Explain with AI` button on every card (see **LLM Features**)
- `Continue` button returns to Home

## LLM Features

Two LLM-powered learning utilities are built into existing screens (satisfying the task's "2+" requirement).

### Generate AI Hint — Quiz
- Implementation: [QuizScreen.kt](app/src/main/java/com/example/llm61/screens/QuizScreen.kt) + [QuizViewModel.kt](app/src/main/java/com/example/llm61/viewmodel/QuizViewModel.kt)
- Prompt explicitly instructs Gemini **not to reveal the answer** and to give a 1–2 sentence encouraging hint
- UI states:
  - Idle → outlined `Get AI Hint` button
  - Loading → `Generating hint…` with a spinner
  - Success → collapsible `📝 Prompt sent to AI` card + `💡 AI Hint` response card + `Hide hint`
  - Error → `❌ Couldn't get hint` card + `Try Again`

### Explain with AI — Results
- Implementation: [ResultsScreen.kt](app/src/main/java/com/example/llm61/screens/ResultsScreen.kt) + [ResultsViewModel.kt](app/src/main/java/com/example/llm61/viewmodel/ResultsViewModel.kt)
- Prompt differs by correctness: reinforcement (2–3 sentences) when the student got it right, correction + encouragement (2–3 sentences) when wrong
- Failures mapped to friendly messages for timeouts, HTTP 429 (rate-limited), and offline (`Unable to resolve host`)
- UI states mirror the Hint flow: Idle / Loading (`Generating explanation…`) / Success (`🤖 AI Explanation`) / Error (`❌ Couldn't get explanation`)

### Shared infrastructure
- Sealed UI state [LlmUiState.kt](app/src/main/java/com/example/llm61/network/LlmUiState.kt): `Idle`, `Loading`, `Success(prompt, response)`, `Error(prompt, message)`
- Reusable `PromptAndResponseDisplay` composable in `QuizScreen.kt` renders the collapsible prompt + response card for both utilities
- Network layer: [GeminiService.kt](app/src/main/java/com/example/llm61/network/GeminiService.kt) (Retrofit interface) + [GeminiRepository.kt](app/src/main/java/com/example/llm61/network/GeminiRepository.kt) (`generate(prompt): Result<String>`) + [GeminiModels.kt](app/src/main/java/com/example/llm61/network/GeminiModels.kt)
- Model: `gemini-flash-latest` via `v1beta/models/gemini-flash-latest:generateContent`

## Input & Validation Rules
- **Login**: blank username or password shows an inline error (`Please enter a username` / `Please enter a password`)
- **Signup**: username / email / password must be non-blank; email must match confirm email; password must match confirm password. Errors surface inline on submit (button itself is not disabled). Phone number is collected but not validated.
- **Interests**: `Next` is disabled until at least 1 topic is selected; taps beyond 10 are silently ignored (hard cap enforced in `UserViewModel.toggleInterest`)
- **Quiz**: `Submit` is disabled until all 3 questions are answered

---

## Output Formatting
- **Quiz counter**: `"{n} of 3 answered"` in the bottom bar
- **Score**: `"{score} / {total}"` displayed large on the Results screen with a `Correct Answers` label
- **Per-answer card**: shows `Your answer: …`; for incorrect answers it also shows `Correct answer: …` in green
- **AI Hint card**: response labelled `💡 AI Hint`; prompt labelled `📝 Prompt sent to AI` (collapsible — `Show` / `Hide`)
- **AI Explanation card**: response labelled `🤖 AI Explanation`
- **Loading**: spinner + `Generating hint…` or `Generating explanation…`
- **Error**: red card titled `❌ Couldn't get hint` or `❌ Couldn't get explanation` with the failure message and a `Try Again` button

## UI Notes
- Material 3 theme defined in [Theme.kt](app/src/main/java/com/example/llm61/ui/theme/Theme.kt)
- Cards use `animateContentSize()` so the Hint / Explanation sections expand and collapse smoothly
- Route transitions: 400 ms slide-left + fade-in when navigating forward, slide-right + fade-out on back — defined in [AppNavigation.kt](app/src/main/java/com/example/llm61/AppNavigation.kt)
- `contentDescription` set on Back and Edit icons for screen-reader support
- No dead ends: Results `Continue` returns to Home; Home provides `Logout` and edit-interests; every other screen has a working Back button or text action

## How to Run
1. Clone the repo and open the project in Android Studio (JDK 17, Android SDK 36 installed)
2. Get a Gemini API key from **Google AI Studio**
3. Open [local.properties](local.properties) and add a line:
   ```
   GEMINI_API_KEY=your_key_here
   ```
   (`local.properties` is git-ignored — keys will not be committed)
4. Sync Gradle
5. Run the `app` module on an emulator or physical device with Android 7.0+ (API 24+)
6. The app requires the `INTERNET` permission (already declared in [AndroidManifest.xml](app/src/main/AndroidManifest.xml))
